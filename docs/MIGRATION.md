# Rodauth Migration Guide

This guide explains the transition from the old Devise+JWT authentication system to the new Rodauth-based system.

## What Changed?

### Old System (Devise + devise-jwt)
- User model with Devise modules
- Password stored in `users.encrypted_password`
- JWT generated by devise-jwt gem
- Session middleware mixed with JWT validation
- Brittle test code (session deserialization)

### New System (Rodauth + Rolify)
- User model with Rodauth JWT
- Password stored in separate `accounts.password_hash`
- Dedicated `Account` model for authentication
- Stateless JWT validation
- Role management via Rolify gem (replaces enum)
- Complete separation of Portal (JWT) and Admin (Sessions)

## Key Differences

| Aspect | Old System | New System |
|--------|-----------|-----------|
| **User Model** | Devise `database_authenticatable` | Rodauth `:jwt` |
| **Roles** | Enum field in `users` table | Rolify `has_many :roles` |
| **Password Storage** | `users.encrypted_password` | `accounts.password_hash` |
| **Admin Users** | Mixed with portal users | Separate `AdminUser` model |
| **Token Format** | devise-jwt JWT | Rodauth JWT (same format, cleaner code) |
| **Token Expiration** | 1 day (same) | 1 day (same) |
| **Login Response** | Token in Authorization header | Token in response body |
| **Tests** | Brittle session deserialization | Clean JWT integration tests |

## Upgrade Path (For New Developers)

### For Fresh Installations

Simply follow the [Clean Setup guide](../CLEAN_SETUP.md):

```bash
cd apps/ehr-api

# Uses schema:load (not migrate), creates fresh database with Rodauth tables
bin/rails db:create
bin/rails db:schema:load
bin/rails db:seed  # Creates test users with password='password'
```

No migration history to worry about.

### For Existing Deployments

If your application is already running the old Devise-JWT system:

1. **Backup your database**
   ```bash
   pg_dump ehr_api_production > backup-$(date +%s).sql
   ```

2. **Deploy the new code**
   ```bash
   git pull origin main
   bundle install
   ```

3. **Run migrations**
   ```bash
   # These migrations were already created during development:
   # - Create Rodauth tables (accounts, account_statuses, account_audit_logs)
   # - Create Rolify tables (roles, users_roles)
   # - Migrate data from old schema
   # - Remove old Devise columns

   rails db:migrate
   ```

4. **Verify migration success**
   ```bash
   # Test login with existing credentials
   curl -X POST http://localhost:3000/api/v1/auth/login \
     -H "Content-Type: application/json" \
     -d '{"user": {"email": "provider@ehr.local", "password": "password"}}'

   # Should return: { user: {...}, token: "eyJ..." }
   ```

5. **Monitor for issues**
   - Check application logs for JWT decode errors
   - Monitor user logins for failures
   - Verify GraphQL queries work with new tokens

## Database Migration Details

### Migration Step 1: Create Rodauth Tables

```ruby
# db/migrate/[timestamp]_create_rodauth_tables.rb
create_table :accounts do |t|
  t.references :user, foreign_key: true, null: false
  t.string :email, null: false, index: true
  t.string :password_hash, null: false
  t.string :status, default: "verified", null: false
  t.datetime :last_login_at
  t.string :last_login_ip
  t.integer :failed_login_attempts, default: 0
  t.timestamps
end
```

### Migration Step 2: Create Rolify Tables

```ruby
# db/migrate/[timestamp]_create_rolify_tables.rb
create_table :roles do |t|
  t.string :name, null: false
  t.string :resource_type
  t.bigint :resource_id
  t.timestamps
  t.index [:name, :resource_type, :resource_id], unique: true
end

create_join_table :users, :roles do |t|
  t.index [:user_id, :role_id], unique: true
end
```

### Migration Step 3: Migrate Data

```ruby
# db/migrate/[timestamp]_migrate_users_to_rodauth.rb
class MigrateUsersToRodauth < ActiveRecord::Migration[8.0]
  def up
    # For each existing user, create an Account
    User.find_each do |user|
      # Create account with migrated password hash
      Account.create!(
        user_id: user.id,
        email: user.email,
        password_hash: user.encrypted_password,  # Copy existing bcrypt hash
        status: "verified"
      )

      # Migrate role enum to Rolify
      case user.role
      when "provider"
        user.add_role(:provider)
      when "staff"
        user.add_role(:staff)
      when "patient"
        user.add_role(:patient)
      end
    end
  end

  def down
    Account.delete_all
    UserRole.delete_all
    Role.delete_all
  end
end
```

### Migration Step 4: Clean Up Old Columns

```ruby
# db/migrate/[timestamp]_remove_devise_columns_from_users.rb
class RemoveDeviseColumnsFromUsers < ActiveRecord::Migration[8.0]
  def change
    remove_column :users, :encrypted_password, :string
    remove_column :users, :reset_password_token, :string
    remove_column :users, :reset_password_sent_at, :datetime
    remove_column :users, :remember_created_at, :datetime
    remove_column :users, :role, :string
  end
end
```

## Code Migration Examples

### User Model

**Before (Devise+JWT):**
```ruby
class User < ApplicationRecord
  devise :database_authenticatable, :jwt_authenticatable, :validatable

  enum role: { patient: 0, provider: 1, staff: 2, admin: 3 }

  def jwt_payload
    { sub: id }
  end
end
```

**After (Rodauth+Rolify):**
```ruby
class User < ApplicationRecord
  rodauth :jwt

  has_one :account, dependent: :destroy
  has_many :roles, dependent: :destroy

  validate :has_at_least_one_role

  def has_role?(role_name)
    roles.exists?(name: role_name)
  end

  def add_role(role_name)
    roles.create!(name: role_name) unless has_role?(role_name)
  end

  private

  def has_at_least_one_role
    errors.add(:base, "User must have at least one role") if roles.empty?
  end
end
```

### Login Controller

**Before (Devise+JWT):**
```ruby
class Api::V1::Auth::SessionsController < Api::ApplicationController
  def login
    user = User.find_by(email: user_params[:email])

    if user&.valid_password?(user_params[:password])
      render json: {
        user: UserSerializer.new(user),
        # Token returned in Authorization header (confusing)
      }, headers: { "Authorization": generate_jwt_token(user) }
    else
      render json: { error: "Invalid credentials" }, status: :unauthorized
    end
  end

  private

  def generate_jwt_token(user)
    # devise-jwt config
    JwtManager.encode(sub: user.id)
  end
end
```

**After (Rodauth):**
```ruby
class Api::V1::Auth::SessionsController < Api::ApplicationController
  def login
    user = User.find_by(email: user_params[:email])

    if user&.account&.valid_password?(user_params[:password])
      token = generate_jwt_token(user)
      render json: {
        user: {
          id: user.id,
          email: user.email,
          roles: user.roles.pluck(:name)
        },
        token: token  # Token in response body (standard)
      }
    else
      render json: { error: "Invalid credentials" }, status: :unauthorized
    end
  end

  private

  def generate_jwt_token(user)
    secret = Rails.application.credentials.secret_key_base
    payload = {
      sub: user.id,
      exp: 1.day.from_now.to_i,
      iat: Time.now.to_i,
      iss: "ehr-portal-api"
    }
    JWT.encode(payload, secret, "HS256")
  end
end
```

### GraphQL Authentication

**Before (Devise+JWT):**
```ruby
class GraphqlController < ApplicationController
  before_action :authenticate_api_user!

  def execute
    # Devise middleware handles authentication
    result = EhrApiSchema.execute(
      query,
      variables: variables,
      context: { current_user: current_user }
    )
    render json: result
  end

  private

  def current_user
    # Devise provides this via middleware
    @current_user ||= warden.user
  end
end
```

**After (Rodauth):**
```ruby
class GraphqlController < ActionController::API
  include ActionController::Cookies
  protect_from_forgery with: :null_session
  before_action :authenticate_graphql_user!

  def execute
    result = EhrApiSchema.execute(
      query,
      variables: variables,
      context: { current_user: current_user }
    )
    render json: result
  end

  private

  def current_user
    @current_user ||= load_user_from_jwt_token || load_admin_from_session
  end

  def load_user_from_jwt_token
    token = extract_token_from_request
    return nil unless token

    begin
      secret = Rails.application.credentials.secret_key_base
      payload = JWT.decode(token, secret, true, { algorithm: "HS256" }).first
      user_id = payload["sub"]&.to_i
      user = User.find_by(id: user_id)
      user if user && user.account&.status == "verified"
    rescue JWT::DecodeError, JWT::ExpiredSignature
      nil
    end
  end

  def extract_token_from_request
    request.headers["Authorization"]&.sub(/\ABearer\s+/, "")
  end

  def authenticate_graphql_user!
    render json: { error: "Unauthorized" }, status: :unauthorized unless current_user
  end
end
```

### Spec Tests

**Before (Devise+JWT - Brittle):**
```ruby
require 'rails_helper'

describe Api::V1::UsersController do
  describe "GET /api/v1/users/:id" do
    it "returns the user when authenticated" do
      user = create(:user, :provider)

      # Brittle: manually setting up session
      allow_any_instance_of(Warden::Proxy).to receive(:user).and_return(user)

      get "/api/v1/users/#{user.id}"
      expect(response).to have_http_status(:ok)
      expect(json_response["id"]).to eq(user.id)
    end
  end
end
```

**After (Rodauth - Clean):**
```ruby
require 'rails_helper'

describe Api::V1::UsersController do
  describe "GET /api/v1/users/:id" do
    it "returns the user when authenticated" do
      user = create(:user, :provider)
      token = login_user(user)  # Helper function

      get "/api/v1/users/#{user.id}",
          headers: { "Authorization": "Bearer #{token}" }

      expect(response).to have_http_status(:ok)
      expect(json_response["id"]).to eq(user.id)
    end
  end
end

# Helper in spec/support/auth_helper.rb
def login_user(user, password = "password")
  post "/api/v1/auth/login", params: {
    user: { email: user.email, password: password }
  }
  JSON.parse(response.body)["token"]
end
```

### Portal Client Code

**Before (Devise+JWT - Reading from header):**
```typescript
// apps/ehr-portal/src/lib/auth.ts
async function login(email: string, password: string) {
  const response = await fetch(`${apiUrl}/api/v1/auth/login`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ user: { email, password } })
  });

  // Old: Token came in Authorization response header
  const token = response.headers.get("Authorization")?.replace("Bearer ", "");
  localStorage.setItem("auth_token", token);
  return await response.json();
}
```

**After (Rodauth - Reading from body):**
```typescript
// apps/ehr-portal/src/lib/auth.ts
async function login(email: string, password: string) {
  const response = await fetch(`${apiUrl}/api/v1/auth/login`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ user: { email, password } })
  });

  // New: Token comes in response body
  const data = await response.json();
  localStorage.setItem("auth_token", data.token);
  localStorage.setItem("current_user", JSON.stringify(data.user));
  return data.user;
}
```

**Token Usage (No Change):**
```typescript
// Both old and new send token the same way
const headers = {
  "Authorization": `Bearer ${localStorage.getItem("auth_token")}`
};

// GraphQL query
const response = await fetch(`${apiUrl}/graphql`, {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    ...headers
  },
  body: JSON.stringify({ query, variables })
});
```

## Troubleshooting Migration

### Issue: Users can't login after migration

**Symptoms:** `Invalid credentials` error even with correct password

**Causes:**
1. Password hash not copied correctly
2. Account not created for user
3. Role not assigned (validation fails)

**Solution:**
```bash
# Check that accounts exist
rails console
> User.all.each { |u| puts "#{u.email}: #{u.account.present?}" }

# Check that all users have roles
> User.all.each { |u| puts "#{u.email}: #{u.roles.pluck(:name)}" }

# Manually fix if needed
> user = User.find_by(email: "provider@ehr.local")
> user.create_account!(password: "password") unless user.account
> user.add_role(:provider) unless user.has_role?(:provider)
```

### Issue: GraphQL queries return "Unauthorized"

**Symptoms:** All authenticated GraphQL queries fail with 401

**Causes:**
1. Token not in Authorization header
2. Token format incorrect
3. JWT secret key mismatch

**Solution:**
```bash
# Check token format
rails console
> token = User.first.generate_jwt_token
> payload = JWT.decode(token, Rails.application.credentials.secret_key_base)
> puts payload.inspect  # Should have sub, exp, iat, iss

# Test GraphQL request manually
curl -X POST http://localhost:3000/graphql \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN_HERE" \
  -d '{"query": "{ currentUser { id } }"}'
```

### Issue: Tests failing with "frozen Array"

**Symptoms:** `FrozenError: can't modify frozen Array` in tests

**Causes:**
- Stale database fixtures
- Role validation on frozen objects

**Solution:**
```bash
# Clean database before running tests
bin/rails db:prepare
bin/rspec
```

### Issue: ActiveAdmin not working

**Symptoms:** Redirect loop at /admin/login

**Causes:**
1. AdminUser migration didn't create any users
2. Devise not configured for AdminUser model
3. ActiveAdmin config pointing to wrong model

**Solution:**
```bash
# Create admin user manually
rails console
> AdminUser.create!(email: "admin@ehr.local", password: "change_me!")

# Verify ActiveAdmin routes
> Rails.application.routes.routes.select { |r| r.path =~ /admin/ }
```

## Rollback (If Needed)

If migration fails and you need to rollback:

```bash
# Rollback last 4 migrations
rails db:rollback STEP=4

# Or rollback to a specific version
rails db:migrate VERSION=20260321120000

# Verify you're back to old schema
rails db:schema:dump  # Check schema.rb
```

## Performance Impact

- **Login**: Slightly faster (dedicated Account table vs User table)
- **GraphQL queries**: Same speed (JWT verification is <1ms)
- **Password verification**: ~250ms (bcrypt cost factor 12, intentional)

No significant performance changes expected.

## Security Improvements

✅ **Better separation of concerns**
- Portal auth isolated from admin auth
- Rodauth is modern and maintained

✅ **Audit logging built-in**
- Account status changes tracked
- Preparation for HIPAA compliance logging

✅ **Cleaner code**
- Less magic (no session deserialization)
- Easier to test and reason about
- Better TypeScript types
