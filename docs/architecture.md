### Observer Trigger

When a patient's `InsuranceProfile` is created, `InsuranceProfileObserver#after_create`
automatically creates an `InsuranceVerification`, enqueues it, and dispatches the worker вҖ”
no explicit controller call required.

    InsuranceProfile.create!
        вҶ’ InsuranceProfileObserver#after_create
        вҶ’ InsuranceVerification.create! + enqueue! + broadcast!
        вҶ’ InsuranceVerificationWorker.perform_async

### RTE State Machine

    pending вҶ’ queued вҶ’ requesting вҶ’ verified
                     вҶҳ            вҶ— (cache hit: queued вҶ’ verified directly)
                       вҶ’ failed
    verified вҶ’ expired
    pending/queued вҶ’ canceled
